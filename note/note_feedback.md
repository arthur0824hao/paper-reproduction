# Note Feedback

## Current Friction

1. `note/note_tasks.md` referenced `note_rules.md`, `note_schema.md`, and `note_feedback.md` before those files existed.
2. Push state in integrator metadata drifted after remote configuration changed.
3. The round bundle is strong on deliverables but weaker on process documentation for future sessions.
4. Review-agent inbox instructions have now moved into `review/REQUEST_BUNDLE.md`, so protocol docs must point to the new source.

## Recommended Improvements

1. Keep bootstrap note files versioned together with the task board.
2. Treat remote-configuration changes as a mandatory post-closeout metadata sync.
3. When a new collaboration pattern stabilizes, record it in rules/schema before the next round starts.
4. Prefer `review/REQUEST_BUNDLE.md` as the execution contract and keep `note/note_tasks.md` as the readonly user log.

## Handoff Reminder

- Review agent should validate protocol files as guidance, not as retroactive hard constraints for already-closed tickets.
- When `review/REQUEST_BUNDLE.md` exists, use it as the first source for ticket claiming and required bundle sections.
